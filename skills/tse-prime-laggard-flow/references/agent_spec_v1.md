# 投資判断エージェント仕様 v1

## 1. 目的

東証プライム上場銘柄から「出遅れ株候補」を抽出し、定量・定性の両面で評価して、最終的に上位3件の投資判断（推薦のみ）を提示する。

## 2. 対象と前提

- 対象市場: 東証プライム
- J-Quants連携: V2 API（`x-api-key` 認証）
- 基本流動性条件: 直近N営業日平均売買代金が5億円以上
  - Nは `JQUANTS_LIQUIDITY_LOOKBACK_DAYS` で指定（デフォルト5）
- 補助条件: 時価総額が300億円以上（`JQUANTS_ENFORCE_MARKET_CAP=1` のときのみ適用）
- 取引執行は行わない（推薦のみ）
- 無料優先モードではデータ鮮度制約を明示する
- 無料優先モードではJ-Quants提供期間内の `as-of` を指定する

## 3. 評価フロー

1. 定量スクリーニングで候補20件を抽出
2. 候補20件に対して定性5軸で採点
3. 合計点で並べ替え、上位3件を詳細調査
4. 条件付きで投資推薦を出力
5. 各推薦銘柄について、具体的な出遅れ原因を明示する
6. 各投資判断に対して、批判的な意見（反対論点）を併記する
7. 対象業種の景気動向・構造トレンドを整理し、評価に反映する
8. 同業種内での当該企業の強み・弱みを比較評価する

## 4. 定量評価（実装準拠）

実装では以下の2段階で扱う。

1. 戦略指標（設定ファイル）  
`config/strategy_v1.yaml` の10指標を評価対象とする
2. 実データで算出可能な指標  
現行のJ-Quants V2実装で定量スコアに反映される主な指標は次の6つ
   - PBR
   - PER
   - ROE
   - 配当利回り
   - 営業CFマージン
   - 自己資本比率

未取得指標（例: EV/EBITDA, 3年CAGR, ネットD/E）は `v1` では未反映とし、後続実装で拡張する。

### 4.1 スコアリング方針（v1）

- v1は重みなし（等重み）で運用
- 指標ごとにスコアを0-100へ変換
- 定量は2系統で計算する
  - `price_now`: 価格連動（PBR, PER, 配当利回り）
  - `fundamentals_base`: 財務連動（ROE, CF, 自己資本比率ほか）
- 定量総合点は2系統スコアの単純平均（片系統のみ算出できる場合はその値を採用）
- 定量総合点の上位20件を候補とする

## 5. 定性評価（5軸）

各軸を0-5点で採点し、必ず根拠テキスト・参照元URL・取得日付を残す。

1. 出遅れ要因の一時性
2. 成長ドライバーの実現確度
3. 経営品質・資本政策
4. 競争優位性
5. リスク耐性

### 5.1 スコアリング方針（v1）

- v1は重みなし（等重み）
- 定性総合点は5軸の単純合計（0-25点）

## 6. 除外ルール（重大リスク）

以下のいずれかに該当した場合は除外フラグを立てる。

1. 継続企業の前提に関する重要事象（GC疑義）
2. 重大な会計不正・法令違反が直近で確認される
3. 債務超過
4. 営業CFが長期でマイナス傾向
5. 流動性条件を満たさない
6. 直近決算で大幅下方修正かつ説明が不十分

## 7. 最終順位と上位3件選定

- 暫定総合点 = `定量総合点(0-100)` + `定性総合点を100点換算`
- 除外フラグ付き銘柄は上位候補から除外
- 残りを暫定総合点で並べ、上位3件を詳細調査対象にする

## 8. 上位3件の詳細調査

各銘柄について以下を実施する。

1. 直近ニュース確認（推奨: 30日以内を優先）
2. ポジティブ要因とネガティブ要因の整理
3. 反証（買わない理由）の先行検証
4. シナリオ整理（ベース/強気/弱気）
5. 条件付き推薦の結論化
6. 評価文は抽象表現を避け、企業固有の数値・出来事を必ず含める
   - 禁止例: 「業績が予想を上回る場合、再評価の可能性」
   - 禁止例: 「市場全体の景気後退が影響を及ぼす可能性がある」

## 9. 出力フォーマット

### 9.1 候補20件テーブル

必須列:

- ティッカー
- 企業名
- 業種
- 定量総合点
- 定量スコア（`Q(Price)`）
- 定量スコア（`Q(Fund)`）
- 定性5軸スコア
- 定性総合点
- 暫定総合点
- 除外フラグ
- 主根拠リンク
- データ日付

### 9.2 上位3件レポート

必須項目:

- 見出しに `銘柄コード + 企業名 + 業務概要`
- 推薦判断（Recommend / Watch / Skip）
- 推薦理由（3点以内）
- 主要リスク（3点以内）
- 推薦の前提条件
- 破綻シナリオ
- 再評価トリガー
- 根拠リンク（ニュース・決算資料）

### 9.3 付録（上位3件の詳細投資評価）

レポート末尾に、上位3件それぞれの付録評価を追加する。  
各銘柄は `証券コード + 企業名` 見出しで、以下の形式を固定で使う。

1. ① 業績の安定性と成長性
- 定量: 過去5期売上、過去5期営業利益、EPS、直近四半期トレンド、今後3年見通し
- 定性: 成長の質（価格転嫁/数量増、特需依存の有無）
- 評価: `A/B/C/D`

2. ② 財務健全性
- 定量: 自己資本比率、有利子負債、ネットD/E、営業CF/FCF推移、資金繰りリスク
- 評価: `A/B/C/D`

3. ③ 競争優位性（モート）
- 定量/定性: 主力事業、市場シェア、参入障壁、価格決定力、技術/ブランド/ネットワーク効果、競合比較
- 評価: `A/B/C/D`

4. ④ バリュエーション
- 定量: 現在株価、PER/PSR/PBR、同業比較、PEG的整合
- 評価: `割安/妥当/割高`

5. ⑤ 経営陣の質
- 定量/定性: CEO経歴、株式保有、中計達成度、不祥事有無、株主還元方針
- 評価: `A/B/C/D`

6. 総合評価
- `5項目の総合点（100点満点）`
- `投資スタンス（強気/中立/慎重）`
- `主なリスク3点`
- `カタリスト3点`
- `3行結論`

データ欠損時のルール:
- 欠損項目は推測で埋めない。
- `本レポート収集範囲外（要補完）` と明示する。

## 10. 監査ログ要件

- 銘柄ごとに根拠URL、取得日、採点理由を記録
- スコア変更履歴を保持
- 参照不能なリンクを定期検知

## 11. 既知の制約と今後の改善

- v1は等重みであり最適化未実施
- データソース品質差の影響を受ける
- 無料優先モードは鮮度に制約がある（遅延・公開期間）
- 無料枠ではレート制限（429）が発生しやすい
- v2で重み最適化、セクター別補正、バックテストを導入
